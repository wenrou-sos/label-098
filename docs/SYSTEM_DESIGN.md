# 婚恋市场数据分析看板系统 - 设计文档

## 1. 系统概述

### 1.1 项目背景
本系统是一个功能完整的婚恋市场数据分析看板，旨在通过模拟脱敏的婚恋平台用户数据，从多个维度深入分析婚恋市场的结构特征、用户行为模式和匹配规律。

### 1.2 系统目标
- 提供婚恋市场全景数据洞察
- 支持城市人口结构、择偶偏好、内容分析、焦虑指数、匹配规律五大核心分析模块
- 实现交互式可视化展示
- 提供可复用的数据处理和分析框架

### 1.3 技术栈
| 层级 | 技术选型 | 版本 | 说明 |
|------|---------|------|------|
| 前端框架 | React | 18.x | UI组件化开发 |
| 构建工具 | Vite | 5.x | 快速构建与热更新 |
| UI组件库 | Ant Design | 5.x | 企业级UI组件库 |
| 图表库 | ECharts + echarts-for-react | 5.x | 专业数据可视化 |
| 后端框架 | Flask | 3.x | 轻量级Python Web框架 |
| 数据处理 | Pandas + NumPy | 最新 | 数据分析核心库 |
| 文本分析 | Jieba | 0.42.x | 中文分词引擎 |
| 跨域支持 | flask-cors | 4.x | 前后端跨域通信 |

## 4. 核心算法与分析模型

### 4.1 城市人口结构分析
- **性别比例计算**：男女人口比例 = 男性人数 / 女性人数
- **失衡判定标准**：
  - 男多女少：比例 > 1.1
  - 女多男少：比例 < 0.9
  - 基本均衡：0.9 ≤ 比例 ≤ 1.1
- **年龄分组**：20-25 / 26-30 / 31-35 / 36-40 / 40+

### 4.2 择偶要求差异分析
按年龄段×性别双维度统计四类硬性要求的设定比例：
- 学历硬性要求率
- 收入硬性要求率
- 房产硬性要求率
- 身高硬性要求率

### 4.3 自我介绍内容分析
- **文本分词**：使用 Jieba 中文分词引擎
- **停用词过滤**：过滤助词、代词等无意义词汇
- **Top20关键词提取**：基于词频统计
- **关注点差异对比**：统计35+关键词的提及率

### 4.4 婚恋焦虑指数模型
**综合评分公式**：
```
焦虑指数 = 加权综合得分 / 理论最大值 × 100

其中加权综合得分 =
  付费率 × 0.35 +
  (平均客单价 / 50) +   // 归一化处理
  30岁以上占比 × 0.25 +
  无房占比 × 0.20 +
  低收入占比 × 0.15

焦虑等级判定：
  ≥ 75分 → 极度焦虑
  55-74 → 高度焦虑
  35-54 → 中度焦虑
  < 35 → 轻度焦虑
```

### 4.5 脱单成功规律分析
- **年龄差分布**：8区间分组统计
- **学历/收入差分布**：5区间（女方高2级+ → 男方高2级+）
- **组合热力图**：学历/收入 5×5 交叉矩阵
- **核心规律指标**：
  - 男方年龄偏大占比
  - 年龄差3岁内占比
  - 学历相同/相近占比
  - 收入相同/男方更高占比

## 5. API接口设计

| 方法 | 路径 | 说明 | 返回值 |
|------|------|------|--------|
| GET | /api/health | 服务健康检查 | {status, message} |
| POST | /api/regenerate | 重新生成模拟数据 | {status, message} |
| GET | /api/summary | 获取整体统计概览 | 核心指标汇总 |
| GET | /api/city-demographics?city= | 城市人口结构分析 | 各城市男女比例+年龄分布 |
| GET | /api/preference-differences | 择偶要求差异分析 | 年龄段×性别×要求类型统计 |
| GET | /api/self-intro-analysis | 自我介绍内容分析 | 关键词+关注点提及率 |
| GET | /api/anxiety-index | 婚恋焦虑指数 | 各线城市焦虑评分+付费数据 |
| GET | /api/match-success | 匹配成功规律 | 年龄差/学历差/收入差分布 |
| GET | /api/all-data | 获取全部分析结果 | 以上所有数据汇总 |
